# Interview Guide

Structure candidate evaluation so the right signals get captured every time. Consistent questions and clear rubrics reduce bias and make debrief conversations productive.

## Workflow

1. Gather role context: title, level, key responsibilities, must-have vs. nice-to-have signals
2. Identify the interview format: rounds, panel composition, length, and type (technical / behavioral / cultural)
3. Generate question banks mapped to competencies
4. Draft the full interview plan with time allocations and logistics
5. Create the debrief template for post-interview alignment

## Competency Framework

Map questions to these competency categories based on role:

| Category | Focus | Applies To |
|----------|-------|-----------|
| **Technical depth** | Domain knowledge, problem-solving, code quality | Engineering, data, design |
| **Execution** | Delivering results, prioritization, quality | All roles |
| **Communication** | Clarity, persuasion, documentation | All roles |
| **Collaboration** | Cross-functional work, conflict, teamwork | All roles |
| **Leadership** | Influence, mentorship, decision ownership | Senior+ |
| **Customer empathy** | User understanding, feedback loop | Product, CS, sales |
| **Systems thinking** | Scaling, tradeoffs, long-term thinking | Senior+ |

## Interview Plan Template

```markdown
# Interview Guide: [Role Title] — [Level]
**Rounds:** [List of rounds and interviewers]
**Total duration:** [Hours]
**Hiring goal:** [What specifically we're trying to validate]

---

## Round: [Round Name] — [Interviewer] — [Duration]
**Focus:** [Competencies this round covers]

### Opening (5 min)
- Brief intro and logistics
- "Tell me about your background — what led you here?"

### Core Questions (40 min)
**[Competency 1]:**
- [Primary question]
- Follow-ups: [What does X mean to you?] / [Tell me more about Y] / [What would you do differently?]

**[Competency 2]:**
- [Primary question]
- Follow-ups: [...]

### Candidate Questions (10 min)
- Let candidate ask — note what they ask (signals priorities and preparation)

### Notes field
[Space for interviewer notes during conversation]
```

## Behavioral Question Patterns (STAR)

Structure behavioral questions to elicit Situation → Task → Action → Result responses:
- "Tell me about a time when you had to [challenge]..."
- "Describe a situation where [scenario] — how did you handle it?"
- "What's the most [complex / impactful / difficult] [thing] you've worked on?"
- "Give me an example of when you [disagreed / failed / took ownership]..."

**Good follow-ups for any STAR answer:**
- "What specifically did YOU do?" (if team is over-emphasized)
- "What was the outcome?" (if result wasn't stated)
- "What would you do differently?" (reveals reflection quality)
- "How did others react?" (tests situational awareness)

## Technical Interview Formats

**Coding:** Define language, environment, and difficulty tier. Have 2 problems ready — easy warm-up + harder signal
**System design:** Choose a system relevant to the role. Evaluate: clarifying questions asked, tradeoffs surfaced, communication clarity
**Case study / take-home:** Define time expectation explicitly. Score on approach + output quality + presentation of tradeoffs

- [Specific signal with example]

## Scoring Scale (1–4)

| Score | Meaning |
|-------|---------|
| 4 — Strong hire signal | Exceeds bar; would raise team average |
| 3 — Hire signal | Meets bar; would contribute immediately |
| 2 — Weak signal | Below bar on this dimension; notable gap |
| 1 — No hire signal | Clear miss; fundamental concern |

## Quality Checks

Before the interview:
- Questions map to stated must-have competencies, not interviewers' personal interests
- No questions about protected characteristics (age, family, origin, health)
- Panel is calibrated: each round covers different competencies, no redundant coverage
- Debrief is scheduled before the interview, not improvised afterward
